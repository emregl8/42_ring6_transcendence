=========== İLK KURULUM ===========

1. debian sanal makine kurun (docker engine, debian tabanlı olsa bile farklı dağıtımlarda farklı davranabiliyor. o yüzden debian kullanın.)

2. aşağıdaki paketleri indirin
sudo apt install -y curl wget git openssl make npm

3. aşağıdaki linkten son sürüm docker engine'i kurun (apt ile de indirilebilir ancak apt ile son sürüm inmediğinden sorun çıkabilir)
https://docs.docker.com/engine/install

4. kubectl'yi kurun
https://kubernetes.io/docs/tasks/tools/install-kubectl-linux/

5. aşağıdaki paketi indirin (kind bazen tekte inmiyor. inmezse tekrar deneyin)
sudo apt install -y kind

6. kullanıcınızın konteyner çalıştırabilmesi için gerekli gruplara eklenmesi gerek (sudo olarak çalıştırmayın, gelecekte yetki problemleri yaşarsınız)
sudo usermod -aG docker $USER
üstteki komutu yazdıktan sonra bilgisayarı yeniden başlatmanız gerek

6. transcendence.local adresini /etc/hosts'a ekleyin
echo "127.0.0.1 transcendence.local" | sudo tee -a /etc/hosts

7. "make install-deps" ile gerekli bağımlılıkları indirin.

8. "make dev" ile sunucuyu ayağa kaldırabilirsiniz (cluster yapısı ve teknolojiler aws/digitalocean'a geçiş için ayarlandı. lütfen rastgele değiştirmeyin)
https://transcendence.local adresine giderek siteye erişebilirsiniz.


=========== NOTLAR =========== 

1. "make clean" namespace'i sildiği için volume'ler de silinir ve tüm data(veritabanı) kaybolur.
Data'nın korunması için pod'ları "make rebuild-frontend" veya "make rebuild-postgres" gibi rebuild komutlarıyla yeniden başlatın.

2. Proje her derlendiğinde parolalar otomatik sıfırlanıp .env'e kaydediliyor.

3. Şimdilik POSTGRES_PASSWORD'ü postgres-secret.yaml'e koydum(güvenlik açığı). HashiCorp Vault eklediğimde oraya taşıyacağım.

4. pdb gibi terimleri daha önce görmediyseniz ben anlatırım. ben de araştırırken öğrenip ekledim. proje için kritik şeyler değil.
