=========== İLK KURULUM ===========

1. debian sanal makine kurun 
- docker engine, debian tabanlı olsa bile farklı dağıtımlarda farklı davranabiliyor. o yüzden tercihen debian kullanın.
- makineye az disk ve kaynak vermeyin. pod sayısı fazla olduğundan proje çalışmaz. kaynak sorunu yaşıyorsanız projeyi ana makinede çalıştırın.

2. aşağıdaki paketleri indirin
sudo apt install -y curl wget git openssl make npm

3. aşağıdaki linkten son sürüm docker engine'i kurun (apt ile de indirilebilir ancak apt ile son sürüm inmediğinden sorun çıkabilir)
https://docs.docker.com/engine/install

4. kubectl'yi kurun
https://kubernetes.io/docs/tasks/tools/install-kubectl-linux/

5. aşağıdaki paketi indirin (kind bazen tekte inmiyor. inmezse tekrar deneyin)
sudo apt install -y kind

6. kullanıcınızın konteyner çalıştırabilmesi için gerekli gruplara eklenmesi gerek (sudo olarak çalıştırmayın, gelecekte yetki problemleri yaşarsınız)
sudo usermod -aG docker $USER
üstteki komutu yazdıktan sonra bilgisayarı yeniden başlatmanız gerek

7. transcendence.local adresini /etc/hosts'a ekleyin
echo "127.0.0.1 transcendence.local" | sudo tee -a /etc/hosts

8. "make install-deps" ile gerekli bağımlılıkları indirin.

9. "make dev" ile sunucuyu ayağa kaldırabilirsiniz (cluster yapısı ve teknolojiler aws/digitalocean'a geçiş için ayarlandı. lütfen rastgele değiştirmeyin)
https://transcendence.local adresine giderek siteye erişebilirsiniz.


=========== EKİBE NOTLAR =========== (buraya not bırakabilirsiniz)

1. "make clean" namespace'i sildiği için volume'ler de silinir ve tüm data(veritabanı) kaybolur.
Data'nın korunmasını istiyorsanız pod'ları "make rebuild-frontend" veya "make rebuild-postgres" gibi rebuild komutlarıyla yeniden başlatın.

2. Proje her çalıştırıldığında parolalar rastgele oluşturulup .env'e kaydediliyor.


========== GÜVENLİK/PRODUCTION NOTLARI =========

1. unseal key'ler development için secrets olarak eklendi ve root token silindi. production'da kms/hsm kullanılacak.
2. memory lock ayarlanacak
3. imagePullPolicy değiştirilecek
4. letsencrypt ayarlanacak
5. veritabanı iletişimi için ssl açılacak
